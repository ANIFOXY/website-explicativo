CSS

Principais Propriedades e Conceitos de CSS:

CSS (Cascading Style Sheets) é uma linguagem utilizada para descrever a apresentação de um documento escrito em HTML ou XML. Com CSS, é possível controlar o layout, cores, fontes e outros aspectos visuais da página, permitindo criar designs atraentes e responsivos. Além das propriedades básicas, CSS oferece um vasto conjunto de recursos que podem ser usados para estilizar de maneira sofisticada. A capacidade de criar animações, gradientes e efeitos visuais tornam o CSS uma ferramenta essencial para desenvolvedores web.

  • color: Define a cor do texto em um elemento. Pode ser especificada com nomes de cores, valores hexadecimais, RGB ou HSL.
  • background-color: Define a cor de fundo de um elemento. Assim como a cor do texto, pode ser especificada com diferentes métodos.
  • font-size: Define o tamanho da fonte do texto. Pode ser especificado em pixels (px), em (em), porcentagem (%), entre outros.
  • margin: Define o espaço externo ao redor de um elemento. Pode ser usado para separar elementos uns dos outros.
  • padding: Define o espaço interno entre o conteúdo de um elemento e sua borda. É útil para adicionar espaço ao redor do conteúdo dentro de um elemento.
  • border: Adiciona uma borda ao redor de um elemento. Pode ser configurada com largura, estilo e cor.
  • display: Controla como um elemento é exibido na página. Pode ser configurado como block, inline, flex, grid, entre outros.
  • position: Define o método de posicionamento de um elemento. Pode ser static, relative, absolute, ou fixed.
  • flex: Utilizado em layout de flexbox para definir como os itens dentro de um contêiner flexível devem crescer e encolher.
  • grid: Utilizado em layout de grid para criar layouts baseados em grades, oferecendo um controle mais preciso sobre a estrutura da página.
  • text-align: Define o alinhamento do texto em um elemento. Os valores podem ser left, right, center e justify.
  • overflow: Controla como o conteúdo que excede as dimensões de um elemento é tratado. Pode ser visible, hidden, scroll, ou auto.
  • opacity: Define a transparência de um elemento, variando de 0 (completamente transparente) a 1 (completamente opaco).
  • box-shadow: Adiciona uma sombra ao redor de um elemento, permitindo criar efeitos de profundidade.
  • transition: Permite a alteração suave das propriedades CSS ao longo do tempo, melhorando a experiência do usuário.
  • animation: Permite criar animações complexas, definindo quadros-chave (keyframes) que descrevem a mudança ao longo do tempo.
  • gradients: Usados para criar transições de cor, como gradientes lineares e radiais, que podem ser aplicados a fundos.
  • filter: Aplica efeitos visuais como desfoque, brilho e saturação a elementos.
  • z-index: Controla a sobreposição de elementos, determinando qual elemento deve aparecer na frente ou atrás dos outros.
  • cursor: Altera o tipo de cursor que aparece ao passar o mouse sobre um elemento, como pointer, crosshair, e outros.

JavaScript no Backend

Funções e Principais Utilizações no Backend

JavaScript no backend é amplamente utilizado para construir aplicações robustas e escaláveis. As tecnologias associadas ao JavaScript no backend permitem a criação de servidores, APIs e manipulação de bancos de dados, oferecendo uma experiência de desenvolvimento unificada.

Principais Tecnologias e Conceitos

    Frameworks de Aplicações:

    Existem vários frameworks que utilizam JavaScript, além do Node.js, para construir aplicações. Exemplos incluem:

  • AdonisJS: Um framework MVC (Model-View-Controller) para construir aplicações web de forma organizada e eficiente.
  • Sails.js: Um framework MVC que se destaca na construção de APIs e aplicações em tempo real.
  • Express.js: Um framework minimalista e flexível para Node.js, usado para construir servidores e APIs de forma rápida e eficiente. Sua simplicidade e leveza o tornam uma escolha popular para projetos de todos os tamanhos, especialmente para criar APIs RESTful.
  • Microservices: JavaScript é utilizado na construção de arquiteturas de microserviços, onde a aplicação é dividida em pequenos serviços independentes. Cada microserviço pode ser desenvolvido e implantado separadamente, facilitando a escalabilidade e a manutenção.
  • APIs GraphQL: GraphQL é uma alternativa ao REST, permitindo que os clientes solicitem exatamente os dados de que precisam. Com bibliotecas como Apollo Server, você pode implementar APIs GraphQL em JavaScript, proporcionando uma comunicação mais eficiente entre o cliente e o servidor.
  • Banco de Dados: JavaScript é frequentemente usado em conjunto com bancos de dados NoSQL, como MongoDB, que permite armazenar dados em um formato JSON flexível. Além disso, pode ser usado com bancos de dados SQL, utilizando ORMs como Sequelize e TypeORM.
  • Testes Automatizados: O uso de JavaScript no backend permite a implementação de testes automatizados com ferramentas como Mocha, Chai e Jest. Esses testes garantem que a lógica de negócios funcione conforme o esperado e ajudam a prevenir regressões.
  • Segurança: A segurança em aplicações backend é crucial. Implementações de autenticação e autorização com bibliotecas como Passport.js e o uso de JSON Web Tokens (JWT) para gerenciar sessões são práticas comuns para garantir a segurança das aplicações.
  • Serviços em Nuvem: O uso de serviços em nuvem, como AWS Lambda ou Google Cloud Functions, permite que desenvolvedores utilizem JavaScript para construir funções serverless, que são executadas em resposta a eventos, reduzindo a necessidade de gerenciamento de servidores.
  • Integração com Outras Tecnologias: JavaScript pode ser integrado com outras linguagens e tecnologias. Por exemplo, você pode usar JavaScript para orquestrar serviços escritos em Python ou Java, facilitando a comunicação entre diferentes partes da aplicação.

Node.js

O que é Node.js?

Node.js é uma plataforma de desenvolvimento que permite executar código JavaScript no lado do servidor rapida e eficiente. O principal diferencial que o Node.js fornece é o modelo de I/O que não bloqueia e é orientado a eventos, que permite que aplicações realizem varias operações simultaneamente sem atrapalhar no seu desempenho.

Principais Características do Node.js

  • Assíncrono e Orientado a Eventos: Node.js utiliza um modelo de programação assíncrono, onde as operações de entrada e saída (I/O) são realizadas de forma não bloqueante. Isso significa que, enquanto uma operação está aguardando por uma resposta (como uma consulta ao banco de dados), outras operações podem continuar a ser executadas, aumentando a eficiência do código. O uso de callbacks e promessas facilita o gerenciamento de operações assíncronas.
  • Single Thread: Embora o Node.js seja single-threaded, ele pode lidar com muitas conexões simultâneas. Isso é possível graças ao loop de eventos, que gerencia as requisições em um único thread, permitindo que a aplicação continue respondendo a novas requisições enquanto aguarda o término de operações de I/O.
  • NPM (Node Package Manager): O NPM é um poderoso gerenciador de pacotes que facilita a instalação, atualização e gerenciamento de bibliotecas e dependências para projetos Node.js. Com milhões de pacotes disponíveis, os desenvolvedores podem encontrar soluções prontas para diversas necessidades, economizando tempo e esforço no desenvolvimento.
  • Comunidade Ativa: A comunidade de Node.js é uma das mais vibrantes e colaborativas. Isso resulta em um ecossistema rico em pacotes e frameworks, como Express.js, Koa.js e Nest.js, que simplificam a construção de aplicações robustas. A troca constante de conhecimento e a contribuição para projetos open source ajudam a manter a tecnologia atualizada e relevante.
  • Desempenho: O Node.js é conhecido por seu desempenho superior, especialmente em aplicações que exigem alta concorrência e que fazem uso intensivo de operações de I/O. Seu modelo não bloqueante permite que aplicações escalem facilmente e suportem muitas conexões simultâneas sem deteriorar o tempo de resposta.

Principais Casos de Uso do Node.js

  • APIs RESTful: O Node.js é frequentemente utilizado para construir APIs RESTful, que servem como intermediárias entre front-end e back-end. As APIs permitem que aplicações web e móveis comuniquem-se eficientemente, trocando dados em tempo real. O framework Express.js, por exemplo, torna a construção de APIs mais rápida e intuitiva.
  • Aplicações em Tempo Real: Com sua natureza assíncrona e suporte a WebSockets, o Node.js é ideal para aplicações que requerem comunicação em tempo real, como chats, jogos online e plataformas de colaboração. Essas aplicações podem enviar e receber dados instantaneamente, proporcionando uma experiência interativa aos usuários.
  • Microserviços: A arquitetura de microserviços é uma abordagem em que uma aplicação é dividida em serviços menores e independentes. O Node.js é uma escolha popular para essa arquitetura, pois permite que cada microserviço seja desenvolvido, testado e implantado separadamente, facilitando a escalabilidade e a manutenção.
  • Serviços em Nuvem: O Node.js é amplamente utilizado para criar funções serverless em plataformas de nuvem, como AWS Lambda e Google Cloud Functions. Essas funções são executadas em resposta a eventos, eliminando a necessidade de gerenciamento de servidores e permitindo que desenvolvedores se concentrem na lógica de negócios.

React no Frontend

Estruturas, Componentes e Como São Utilizados

React é uma biblioteca JavaScript para criar interfaces de usuário. Ele permite dividir a interface em componentes reutilizáveis e gerenciar o estado da aplicação.

Principais Conceitos

  • Componentes: Em React, componentes são blocos de construção que representam partes da interface. Eles podem ser funcionais (escritos como funções) ou de classe.

    Exemplo de componente funcional:

    function Welcome() { return <h1>Bem-vindo ao site!</h1>; }
  • JSX: JSX é uma extensão de sintaxe que permite escrever HTML dentro do JavaScript. Ele facilita a criação de interfaces React de forma mais clara e intuitiva.

    Exemplo de JSX:

    const elemento = <h1>Olá, Mundo!</h1>;
  • Props: As props (abreviação de "propriedades") são valores passados para os componentes, permitindo que eles sejam reutilizados com diferentes dados.

    Exemplo de uso de props:

    function Usuario(props) { return <p>Nome: {props.nome}</p>; }
  • Estado (State): O estado armazena dados internos do componente que podem mudar ao longo do tempo, permitindo atualizar a interface sem recarregar a página.

    Exemplo de uso de estado:

    const [contador, setContador] = useState(0);
  • Eventos: Eventos em React são escritos em camelCase, como onClick e onChange. Eles permitem responder às ações do usuário.

    Exemplo de evento:

    <button onClick={() => alert('Clique!')}>Clique aqui</button>
  • Renderização Condicional: A renderização condicional permite mostrar diferentes elementos com base no estado ou props.

    Exemplo:

    {isLoggedIn ? <p>Bem-vindo!</p> : <p>Faça login.</p>}
  • Listas e Chaves: Em React, listas são renderizadas usando map(). Cada item deve ter uma key única.

    Exemplo de lista:

    {itens.map((item) => <li key={item.id}>{item.texto}</li>)}
  • useEffect: O hook useEffect é usado para executar efeitos colaterais, como buscar dados, após a renderização.

    Exemplo de uso:

    useEffect(() => { console.log("Componente montado"); }, []);
  • useContext: O useContext permite compartilhar dados entre componentes sem precisar passar props manualmente.

    Exemplo de uso:

    const valor = useContext(Contexto);
  • useRef: useRef cria uma referência mutável para armazenar um valor ou referenciar um elemento do DOM.

    Exemplo de uso:

    const inputRef = useRef(null);
  • Fragmentos: Fragmentos permitem agrupar elementos sem adicionar um elemento extra no DOM.

    Exemplo:

    <> <p>Texto 1</p> <p>Texto 2</p> </>

DevOps (GHPages e PM2/Docker)

DevOps é uma metodologia que une desenvolvimento (Dev) e operações (Ops) com o objetivo de automatizar e melhorar os processos de entrega de software. Ele promove a colaboração entre desenvolvedores e administradores de sistemas, utilizando ferramentas e práticas que garantem entregas rápidas, confiáveis e contínuas. Principais conceitos de DevOps incluem integração contínua (CI), entrega contínua (CD), automação de testes e monitoramento constante. Ferramentas como GitHub Pages, PM2 e Docker desempenham papéis fundamentais nesse ciclo.

GitHub Pages

GitHub Pages é uma solução de hospedagem gratuita oferecida pelo GitHub para sites estáticos. É amplamente utilizada para hospedar páginas de documentação, portfólios e projetos front-end. Os arquivos HTML, CSS e JavaScript são armazenados em um repositório GitHub e são automaticamente servidos como um site público.

Hospedagem e Custo: O serviço é gratuito, mas limitado a sites estáticos. Não suporta servidores backend ou bancos de dados.

Domínio: GitHub Pages permite associar domínios personalizados (ex: www.meusite.com). Basta configurar os registros DNS no provedor de domínio.

Servidores: Os sites são hospedados nos servidores do GitHub, garantindo alta disponibilidade, segurança e distribuição global de conteúdo.

PM2 para Gerenciamento de Processos Node.js

PM2 é um gerenciador de processos para aplicações Node.js. Ele facilita a manutenção de servidores em produção, garantindo que as aplicações continuem rodando mesmo após falhas. PM2 oferece recursos como:

  • Reinício automático: Se o processo falhar, PM2 o reinicia automaticamente.
  • Monitoramento em tempo real: Permite acompanhar logs, uso de CPU e memória.
  • Cluster Mode: Balanceamento de carga distribuindo múltiplas instâncias de uma aplicação.

Hospedagem e Custo: PM2 em si é gratuito e open-source, mas pode ser combinado com qualquer servidor dedicado ou em nuvem, com custos variando conforme a infraestrutura utilizada.

Dockerização de um Serviço

Docker é uma plataforma que permite criar, distribuir e rodar aplicações em contêineres. A dockerização de um serviço significa empacotar a aplicação com todas as suas dependências, de forma que ela possa ser executada de forma consistente em diferentes ambientes, sem a preocupação de “funcionar na minha máquina”. A seguir, vamos ver como dockerizar um serviço Node.js.

Passos para Dockerizar um Serviço Node.js

Para dockerizar um serviço Node.js, você precisa criar um Dockerfile e utilizar o comando docker build para criar a imagem e docker run para rodar o contêiner.

Exemplo de Dockerfile:

        # Definir a imagem base
        FROM node:14
        
        # Definir o diretório de trabalho no contêiner
        WORKDIR /app
        
        # Copiar os arquivos da aplicação
        COPY . .
        
        # Instalar as dependências
        RUN npm install
        
        # Expor a porta onde a aplicação vai rodar
        EXPOSE 3000
        
        # Comando para iniciar a aplicação
        CMD ["npm", "start"]

Este Dockerfile faz o seguinte:

  • Usa a imagem oficial do Node.js como base.
  • Define o diretório de trabalho dentro do contêiner como /app.
  • Copia todos os arquivos do diretório atual (onde o Dockerfile está localizado) para dentro do contêiner.
  • Instala as dependências definidas no package.json.
  • Expondo a porta 3000, onde a aplicação estará acessível.
  • Inicia a aplicação com o comando npm start.

Construindo a Imagem Docker

Com o Dockerfile pronto, o próximo passo é construir a imagem Docker. Você pode fazer isso com o comando:

docker build -t nome-da-imagem .

Rodando o Contêiner Docker

Depois de construir a imagem, você pode rodá-la com o comando:

docker run -p 3000:3000 nome-da-imagem

Isso fará o mapeamento da porta 3000 do contêiner para a porta 3000 do seu host, permitindo que você acesse a aplicação em http://localhost:3000.

Esse arquivo define dois serviços: uma aplicação Node.js e um banco de dados PostgreSQL. O serviço app depende do serviço db. O Docker Compose vai garantir que ambos os serviços sejam executados corretamente e conectados.

docker-compose up

Integração Contínua (CI) e Entrega Contínua (CD)

Integração Contínua (CI) é a prática de integrar frequentemente o código em um repositório compartilhado, onde builds e testes são automatizados. Isso ajuda a identificar erros rapidamente e melhora a qualidade do código.

  • Build: Processo de compilar o código e preparar o ambiente.
  • Testes Automatizados: Garantem que novas alterações não quebrem funcionalidades existentes.
  • Linting e Formatação: Ferramentas como ESLint e Prettier garantem padrões consistentes no código.

Entrega Contínua (Continuous Delivery): O código é mantido sempre em um estado pronto para produção. O deploy é feito manualmente após a aprovação.

Deployment Contínuo (Continuous Deployment): As alterações são automaticamente implantadas em produção após a aprovação dos testes.

Render

Render é uma plataforma de hospedagem em nuvem que facilita o deploy de aplicações modernas. É uma alternativa popular ao Heroku, oferecendo suporte a aplicações estáticas e dinâmicas, APIs e contêineres Docker.

  • Deploy Automático: Render permite deploy diretamente de repositórios GitHub ou GitLab.
  • Suporte a Docker: Aplicações containerizadas podem ser facilmente implantadas.
  • CI/CD Integrado: A plataforma suporta pipelines de CI/CD nativos.
  • Custo: Render oferece planos gratuitos para pequenas aplicações, com opções pagas para mais recursos e performance.

Com Render, desenvolvedores podem criar e escalar aplicações rapidamente, com funcionalidades como domínio personalizado, certificados SSL gratuitos e monitoramento contínuo.
